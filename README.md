# Домоуправител - Система за управление на блок

PWA приложение за управление на блок с React, TypeScript и Supabase.

## Функционалности

- ✅ **Етажна собственост** - Управление на етажи, апартаменти и собственици
- ✅ **Приходи** - Отчитане и управление на приходи
- ✅ **Разходи** - Отчитане и управление на разходи с категории
- ✅ **Документи** - Качване и управление на документи (снимки и PDF)
- ✅ **Потребителски роли** - Администратор, Редактор и Преглед
- ✅ **PWA** - Прогресивно уеб приложение, работи офлайн

## Технологии

- React 18 с TypeScript
- Vite за build tool
- Supabase за база данни и автентикация
- PWA с service worker
- Lucide React за икони
- date-fns за форматиране на дати

## Инсталация

1. Инсталирайте зависимостите:
```bash
npm install
```

2. Настройте базата данни в Supabase:
   - Отворете вашия Supabase проект
   - Отидете в SQL Editor
   - Изпълнете SQL скрипта от `database.sql`

3. Създайте Storage bucket:
   - В Supabase Dashboard отидете в Storage
   - Създайте bucket с име `documents`
   - Направете го публичен (public)

4. Стартирайте development сървъра:
```bash
npm run dev
```

5. Build за production:
```bash
npm run build
```

## Структура на базата данни

### Таблици:
- `users` - Потребители с роли (admin, editor, viewer)
- `floors` - Етажи и собственици
- `income` - Приходи
- `expenses` - Разходи
- `documents` - Документи и файлове

### Роли:
- **admin** - Пълен достъп (създаване, редактиране, изтриване)
- **editor** - Може да създава и редактира данни
- **viewer** - Само преглед, без права за промяна

## Използване

1. Регистрирайте първия потребител като администратор
2. След това можете да създавате други потребители с различни роли
3. Добавете етажи и собственици
4. Запишете приходи и разходи
5. Качете документи и снимки

## PWA Функционалности

Приложението е конфигурирано като PWA и може да се инсталира на устройства. Service worker автоматично кешира ресурсите за офлайн работа.

## Конфигурация

Supabase URL и ключовете са конфигурирани в `src/lib/supabase.ts`. За production използвайте environment variables.

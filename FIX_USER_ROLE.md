# Поправяне на ролята на потребител

Ако вашият потребител има роля "viewer" вместо "admin", ето как да го поправите:

## Вариант 1: Чрез Supabase Dashboard

1. Отворете Supabase Dashboard
2. Отидете в **Table Editor** → **users**
3. Намерете вашия потребител (по email или ID)
4. Редактирайте полето `role` от `viewer` на `admin`
5. Запазете промените

## Вариант 2: Чрез SQL

1. Отворете Supabase Dashboard → **SQL Editor**
2. Изпълнете следния SQL (заменете email-а с вашия):

```sql
UPDATE users 
SET role = 'admin' 
WHERE email = 'admin@abv.bg';
```

## Вариант 3: Обновете RLS политиката

Изпълнете обновения SQL скрипт от `database.sql`, който вече позволява на потребителите да обновяват собствената си роля.

## Проверка

След поправката, излезте и влезте отново в системата. Ролята трябва да е "admin".

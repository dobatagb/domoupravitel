# Връзки между единици

## Описание

Системата поддържа опционални връзки между единици:

### Възможни връзки:
1. **Апартамент ↔ Гараж**
   - Един апартамент може да има свързан гараж
   - Гаражът може да е свързан с апартамент
   - Не е задължително

2. **Апартамент ↔ Паркоместо**
   - Един апартамент може да има свързано паркоместо
   - Паркоместото може да е свързано с апартамент
   - Не е задължително

3. **Гараж ↔ Паркоместо**
   - Гараж може да има свързано паркоместо
   - Не е задължително

### Как работи:

- Полето `linked_unit_id` в таблицата `units` е **опционално** (може да е NULL)
- Когато единица е свързана, `linked_unit_id` съдържа ID-то на свързаната единица
- Връзката е еднопосочна - единицата знае с какво е свързана
- Може да се направи двупосочна връзка чрез UI (когато свързваме А→Б, можем автоматично да свържем Б→А)

### Примери:

**Пример 1: Апартамент с гараж**
- Апартамент №5 → `linked_unit_id` = NULL
- Гараж №3 → `linked_unit_id` = ID на Апартамент №5

**Пример 2: Апартамент с паркоместо**
- Апартамент №10 → `linked_unit_id` = NULL
- Паркоместо №12 → `linked_unit_id` = ID на Апартамент №10

**Пример 3: Самостоятелни единици**
- Апартамент №15 → `linked_unit_id` = NULL
- Гараж №7 → `linked_unit_id` = NULL
- Паркоместо №20 → `linked_unit_id` = NULL

### UI функционалности:

1. При създаване/редактиране на единица:
   - Dropdown за избор на свързана единица (опционално)
   - Филтриране - показване само на подходящи единици (апартаменти за паркоместа, и т.н.)

2. При преглед на единица:
   - Показване на свързаните единици
   - Възможност за премахване на връзката

3. При изтриване:
   - Ако единица се изтрие, `linked_unit_id` на свързаните единици става NULL (ON DELETE SET NULL)

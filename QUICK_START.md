# Бърз старт

## Стъпка 1: Настройка на базата данни

1. Отворете Supabase Dashboard: https://supabase.com/dashboard
2. Изберете вашия проект
3. Отидете в **SQL Editor**
4. Копирайте целия SQL скрипт от `database_v2.sql` (за нов проект). За съществуваща база прилагайте и миграциите от `database_migrations/` по ред.
5. Изпълнете го (кликнете Run)

## Стъпка 2: Настройка на Storage

1. В Supabase Dashboard отидете в **Storage**
2. Кликнете **New bucket**
3. Име: `documents`
4. Направете го **Public** (публичен)
5. Кликнете **Create bucket**

## Стъпка 3: Регистрация на първия потребител

1. Стартирайте приложението: `npm run dev`
2. Отворете http://localhost:3000
3. Кликнете **"Нямате акаунт? Регистрирайте се"**
4. Попълнете:
   - **Имейл**: вашият имейл
   - **Парола**: минимум 6 символа
   - **Роля**: Изберете **"Администратор"**
5. Кликнете **"Регистрация"**

**Важно:** Първият потребител автоматично се създава с роля "viewer", но след това се обновява с избраната от вас роля. Ако има проблем, проверете конзолата на браузъра.

## Стъпка 4: Вход

След успешна регистрация ще влезнете автоматично. Ако не, използвайте същите имейл и парола за вход.

## Решение на проблеми

### Грешка 400 при вход
- Уверете се, че сте се регистрирали първо
- Проверете дали имейлът и паролата са правилни

### Грешка 401 при регистрация
- Уверете се, че сте изпълнили SQL скрипта
- Проверете дали trigger функцията е създадена правилно
- Проверете RLS политиките в Supabase Dashboard

### Не мога да добавя/редактирам данни
- Проверете дали ролята ви е "admin" или "editor"
- Проверете RLS политиките в базата данни

## Принципи

1. **Малки стъпки** — всяка задача да има ясен резултат (merge/deploy).
2. **Схема първо** — при нови модули: таблици + RLS, после UI.
3. **Съвместимост** — да не се чупи съществуващият restore/данни без миграция.
